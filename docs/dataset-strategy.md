# Dataset Audit And Strategy

## Reproducible Audit

Audit date: June 16, 2026.

The local corpus audit is generated by code, not maintained manually. Re-run it
whenever `../Pretraining` changes:

```bash
python scripts/analyze_pretraining_dataset.py \
  --pretraining-root ../Pretraining \
  --dataset-list ../Pretraining/Dataset.txt \
  --output-dir reports/pretraining_dataset_audit
```

Generated artifacts:

- `reports/pretraining_dataset_audit/pretraining_dataset_audit.md`
- `reports/pretraining_dataset_audit/pretraining_dataset_audit.json`
- `reports/pretraining_dataset_audit/pretraining_dataset_audit.csv`

The scanner reads ZIP central directories without extraction. It counts images
by extension, samples a small number of images for mode and band inspection, and
flags containers such as HDF5 without expanding their internal arrays.

## Manifest Coverage

`Pretraining/Dataset.txt` lists 31 public sources. The current downloads match
10 of those source names. `plantnet_300K.zip` is present locally and contributes
most images, but it is not currently listed in `Pretraining/Dataset.txt`; add
its canonical source URL before treating the corpus as publication-ready.

The missing manifest entries include PlantVillage, PlantifyDr, Wheat Plant
Diseases, Plant Pathogen, Weed Detection in Soybean Crops, CottonWeedDet3,
Paddy Doctor, GeoPlant, Pumpkin Leaf Diseases, and several other planned
sources. The generated Markdown report contains the complete missing-source list.

## Current Coverage

Strengths:

- Large RGB image volume, mainly from PlantNet.
- Strong plant species and leaf disease coverage.
- Useful seedling, crop type, rice, chili, medicinal, edible, and toxic plant subsets.
- Class-folder structure is easy to use for SSL pretraining and downstream classification.

Major gaps:

- No local TIFF, GeoTIFF, or NPY multispectral files were found in this audit.
- No downloaded dataset is clearly geospatial, satellite, UAV, NIR, or remote-sensing imagery.
- The corpus is heavily RGB and close-range leaf/species classification oriented.
- Annotation files are sparse; detection, segmentation, and dense prediction are underrepresented.
- There is little explicit metadata for geography, farm, season, crop stage, soil, climate, sensor, or management.
- Duplicate and near-duplicate analysis still needs to be performed before a publication-scale run.
- License, citation, version, and download-date metadata should be frozen per source.

## Priority Additions

1. Real multispectral and georeferenced agricultural imagery with documented bands.
2. Sentinel-2, Landsat, UAV, and field-scale GeoTIFF tiles with RGB, NIR, red-edge, and metadata.
3. Detection and segmentation datasets for weeds, crops, pests, disease lesions, rows, and field boundaries.
4. Multi-season and multi-location acquisitions with farm-held-out and geography-held-out splits.
5. Orchard, fruit, counting, yield, stress severity, and phenotyping datasets.
6. Image-text pairs from expert descriptions, extension documents, and agronomy reports.
7. External datasets reserved exclusively for transfer evaluation.

## Curation Requirements

Before a publication-scale run, create a frozen manifest with:

- source name and canonical URL
- download date and version
- license and redistribution status
- file count, byte size, and format distribution
- channel names and wavelength metadata
- spatial resolution and CRS where applicable
- country, farm, season, crop, and sensor when known
- duplicate and near-duplicate clusters
- train/validation/test assignment
- known label noise and exclusions

## Split Policy

The current loader offers group-disjoint splitting, but the group definition must
be curated. For final evaluation, prefer:

- farm-held-out
- geography-held-out
- season-held-out
- sensor-held-out
- source-dataset-held-out
- crop-held-out

PlantNet's folder structure includes split and species in the group path. It
must be remapped if the scientific split should operate at species, observation,
or geography level.

## Publication Positioning

The defensible contribution is not "many Kaggle datasets combined." It is:

- a provenance-aware, deduplicated, multimodal agricultural corpus
- sensor-adaptive continual pretraining from official ImageNet ViTs
- controlled comparison of reconstruction and self-distillation objectives
- broad transfer under realistic domain shift
- transparent data and evaluation governance
